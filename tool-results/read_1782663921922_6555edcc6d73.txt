     1→     1→# TaxDox AI — Project Worklog
     2→     2→
     3→     3→## Project Overview
     4→     4→TaxDox AI is an AI-native tax document intelligence platform for accounting firms. Built with Next.js 16, TypeScript, Tailwind CSS 4, shadcn/ui, Prisma (SQLite), and z-ai-web-dev-sdk.
     5→     5→
     6→     6→## Tech Stack
     7→     7→- **Framework**: Next.js 16 App Router (single `/` route, client-side view switching)
     8→     8→- **Styling**: Tailwind CSS 4 with custom TaxDox AI teal color palette
     9→     9→- **UI**: shadcn/ui (New York style) + Lucide icons + Recharts
    10→    10→- **Database**: Prisma ORM with SQLite
    11→    11→- **State**: Zustand store at `src/lib/store.ts` for view routing
    12→    12→- **AI**: z-ai-web-dev-sdk for LLM chat (backend only)
    13→    13→
    14→    14→## Architecture
    15→    15→- **Single-page app**: `src/app/page.tsx` switches between view components based on `useAppStore` state
    16→    16→- **Views**: `src/components/views/*.tsx` — each view is self-contained
    17→    17→- **API routes**: `src/app/api/*` — REST endpoints for all CRUD
    18→    18→- **Shared components**: `src/components/shared/` (StatusBadge, PriorityBadge, StatCard, ProgressRing, ConfidenceMeter)
    19→    19→- **Layout**: `src/components/layout/app-shell.tsx` — sidebar + header shell
    20→    20→- **Constants**: `src/lib/constants.ts` — document types, categories, statuses, etc.
    21→    21→- **Types**: `src/lib/types.ts`
    22→    22→
    23→    23→## Navigation
    24→    24→The Zustand store (`useAppStore`) provides:
    25→    25→- `navigate(view)` — switch to a top-level view
    26→    26→- `openEngagement(id)` — open engagement detail
    27→    27→- `openDocument(id)` — open document detail
    28→    28→- `currentView` — current view key
    29→    29→- `selectedEngagementId` / `selectedDocumentId` — selected IDs
    30→    30→
    31→    31→## API Endpoints (all return JSON)
    32→    32→- `GET /api/dashboard` — dashboard stats, recent engagements, team workload, activities
    33→    33→- `GET /api/clients?search=&status=&type=` — list clients (includes _count)
    34→    34→- `POST /api/clients` — create client
    35→    35→- `GET /api/engagements?status=&type=&clientId=` — list engagements (includes client, assignedTo, pbcList, documents, _count)
    36→    36→- `POST /api/engagements` — create engagement
    37→    37→- `GET /api/engagements/[id]` — engagement detail (includes pbcList.items, documents.extractions, workflows, messages, activities)
    38→    38→- `PATCH /api/engagements/[id]` — update engagement
    39→    39→- `POST /api/engagements/[id]/pbc` — generate PBC list from template
    40→    40→- `POST /api/engagements/[id]/send` — send PBC list to client (body: {via})
    41→    41→- `GET /api/documents?engagementId=&clientId=&status=&documentType=` — list documents
    42→    42→- `POST /api/documents` — create/upload document
    43→    43→- `GET /api/documents/[id]` — document detail (includes extractions, client, engagement, pbcItem)
    44→    44→- `PATCH /api/documents/[id]` — update document
    45→    45→- `GET /api/pbc-lists/[id]` — PBC list with items
    46→    46→- `POST /api/pbc-lists/[id]/items` — add PBC item
    47→    47→- `POST /api/ai/classify` — classify document (body: {documentId}) → {documentType, confidence, label, category, fields}
    48→    48→- `POST /api/ai/extract` — extract fields (body: {documentId}) → {extractions}
    49→    49→- `POST /api/ai/chat` — AI chat (body: {messages}) → {reply}
    50→    50→- `GET /api/reports` — operational/financial/quality metrics + trendData + typeDistribution + teamPerformance
    51→    51→- `GET /api/settings/team` — team members
    52→    52→- `GET /api/settings/templates` — PBC templates
    53→    53→- `GET /api/settings/integrations` — tax software + countries
    54→    54→- `GET /api/audit-logs` — audit logs
    55→    55→
