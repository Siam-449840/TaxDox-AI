     1‚Üí# TaxDox AI ‚Äî Project Worklog
     2‚Üí
     3‚Üí## Project Overview
     4‚ÜíTaxDox AI is an AI-native tax document intelligence platform for accounting firms. Built with Next.js 16, TypeScript, Tailwind CSS 4, shadcn/ui, Prisma (SQLite), and z-ai-web-dev-sdk.
     5‚Üí
     6‚Üí## Tech Stack
     7‚Üí- **Framework**: Next.js 16 App Router (single `/` route, client-side view switching)
     8‚Üí- **Styling**: Tailwind CSS 4 with custom TaxDox AI teal color palette
     9‚Üí- **UI**: shadcn/ui (New York style) + Lucide icons + Recharts
    10‚Üí- **Database**: Prisma ORM with SQLite
    11‚Üí- **State**: Zustand store at `src/lib/store.ts` for view routing
    12‚Üí- **AI**: z-ai-web-dev-sdk for LLM chat (backend only)
    13‚Üí
    14‚Üí## Architecture
    15‚Üí- **Single-page app**: `src/app/page.tsx` switches between view components based on `useAppStore` state
    16‚Üí- **Views**: `src/components/views/*.tsx` ‚Äî each view is self-contained
    17‚Üí- **API routes**: `src/app/api/*` ‚Äî REST endpoints for all CRUD
    18‚Üí- **Shared components**: `src/components/shared/` (StatusBadge, PriorityBadge, StatCard, ProgressRing, ConfidenceMeter)
    19‚Üí- **Layout**: `src/components/layout/app-shell.tsx` ‚Äî sidebar + header shell
    20‚Üí- **Constants**: `src/lib/constants.ts` ‚Äî document types, categories, statuses, etc.
    21‚Üí- **Types**: `src/lib/types.ts`
    22‚Üí
    23‚Üí## Navigation
    24‚ÜíThe Zustand store (`useAppStore`) provides:
    25‚Üí- `navigate(view)` ‚Äî switch to a top-level view
    26‚Üí- `openEngagement(id)` ‚Äî open engagement detail
    27‚Üí- `openDocument(id)` ‚Äî open document detail
    28‚Üí- `currentView` ‚Äî current view key
    29‚Üí- `selectedEngagementId` / `selectedDocumentId` ‚Äî selected IDs
    30‚Üí
    31‚Üí## API Endpoints (all return JSON)
    32‚Üí- `GET /api/dashboard` ‚Äî dashboard stats, recent engagements, team workload, activities
    33‚Üí- `GET /api/clients?search=&status=&type=` ‚Äî list clients (includes _count)
    34‚Üí- `POST /api/clients` ‚Äî create client
    35‚Üí- `GET /api/engagements?status=&type=&clientId=` ‚Äî list engagements (includes client, assignedTo, pbcList, documents, _count)
    36‚Üí- `POST /api/engagements` ‚Äî create engagement
    37‚Üí- `GET /api/engagements/[id]` ‚Äî engagement detail (includes pbcList.items, documents.extractions, workflows, messages, activities)
    38‚Üí- `PATCH /api/engagements/[id]` ‚Äî update engagement
    39‚Üí- `POST /api/engagements/[id]/pbc` ‚Äî generate PBC list from template
    40‚Üí- `POST /api/engagements/[id]/send` ‚Äî send PBC list to client (body: {via})
    41‚Üí- `GET /api/documents?engagementId=&clientId=&status=&documentType=` ‚Äî list documents
    42‚Üí- `POST /api/documents` ‚Äî create/upload document
    43‚Üí- `GET /api/documents/[id]` ‚Äî document detail (includes extractions, client, engagement, pbcItem)
    44‚Üí- `PATCH /api/documents/[id]` ‚Äî update document
    45‚Üí- `GET /api/pbc-lists/[id]` ‚Äî PBC list with items
    46‚Üí- `POST /api/pbc-lists/[id]/items` ‚Äî add PBC item
    47‚Üí- `POST /api/ai/classify` ‚Äî classify document (body: {documentId}) ‚Üí {documentType, confidence, label, category, fields}
    48‚Üí- `POST /api/ai/extract` ‚Äî extract fields (body: {documentId}) ‚Üí {extractions}
    49‚Üí- `POST /api/ai/chat` ‚Äî AI chat (body: {messages}) ‚Üí {reply}
    50‚Üí- `GET /api/reports` ‚Äî operational/financial/quality metrics + trendData + typeDistribution + teamPerformance
    51‚Üí- `GET /api/settings/team` ‚Äî team members
    52‚Üí- `GET /api/settings/templates` ‚Äî PBC templates
    53‚Üí- `GET /api/settings/integrations` ‚Äî tax software + countries
    54‚Üí- `GET /api/audit-logs` ‚Äî audit logs
    55‚Üí
