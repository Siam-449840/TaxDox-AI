# TAXDOX AI — Complete Product Specification & Market Research Document

## Executive Summary

TaxDox AI is an end-to-end tax document intelligence platform built for accounting firms, with its core focus on solving the industry's most painful problem: **PBC (Prepared by Client) document management**. There are document processing tools on the market (Magnetic, Black Ore), and there are PBC management tools (Suralink, Byron), but **no product seamlessly integrates the two** — and this is precisely TaxDox AI's differentiating advantage.


## Part One: In-Depth Market Research

### 1.1 Industry Background: The Accounting Industry's Talent Crisis

The accounting industry is facing an unprecedented talent shortage. Since 2021, **300,000 accountants have left the industry**. CPA candidate numbers have been declining consistently, and firms cannot find replacement talent. Meanwhile, across the United States, **80 million individual tax returns are filed through professionals each year**, representing a **$2 billion outsourcing market**, with individual CPA firms paying **$100,000 to $1 million annually** for this service.

> "Accountants don't have a demand problem; they have a capacity problem." — Blaze O'Byrne, Co-founder of Byron

### 1.2 Core Pain Point: The Chaos of PBC Document Management

**Real-world case**: The tax team at JDS Professional Group uses a client portal to send and receive files, but the portal **does not allow preparers to organize received information in a structured way**. Clients send files through various channels including email, USB drives, and even CDs. In large-scale business entities where clients send **over 100 documents**, this lack of organization quickly becomes overwhelming.

> "Clients give us files, but we can't confirm whether this is what we need. It's like a digital shoebox." — Peter Ordelheide, Tax Manager at JDS

### 1.4 Market Gap Analysis

#### Gap 1: Missing Intelligent PBC Layer

Existing PBC management tools (Suralink, AuditDashboard) can only **organize and track** document requests but cannot **understand** document content. AI extraction tools (Magnetic, Black Ore) can extract data but lack PBC workflow management.

**How TaxDox AI fills the gap**: Automatically generate a **data extraction template** for each request item when the PBC request is sent; immediately perform **intelligent validation** upon document upload (checking completeness, correct formatting, and matching the request); and **automatically map** extracted data to corresponding fields in tax software.

#### Gap 2: Affordability Gap for Small and Mid-sized Firms

Black Ore serves Top 100 firms with extremely high pricing. Magnetic and Byron are mid-priced but feature-incomplete. Small firms (1-10 accountants) cannot afford expensive end-to-end solutions.

**How TaxDox AI fills the gap**: Adopts a **pay-as-you-go** model with low starting prices that scale with growth. The MVP can be built using low-code/AI tools within 30-90 days.

#### Gap 3: Missing Cross-Border Tax Support

Existing solutions primarily focus on the U.S. tax system. With **over 190 countries** having different tax systems, multinational enterprises and international individuals urgently need a platform capable of processing tax documents from multiple countries.

**How TaxDox AI fills the gap**: Builds a **configurable tax rules engine** supporting tax system plugins for multiple countries. MVP supports the U.S., UK, Canada, India, and Australia, with expansion capabilities for the future.

#### Gap 4: Client Experience Gap

Existing tools require clients to learn complex portals. Poor client experience leads to delayed document submissions.

**How TaxDox AI fills the gap**: A **zero-learning-curve** client interface — clients simply **drag and drop files or send emails**, and the AI automatically classifies, validates, and matches request items.

## Part Three: Detailed Functional Specifications

### 3.1 Intelligent PBC List Generation Module

#### Function Description
Automatically generates personalized PBC (Prepared by Client) document request lists for each client.

#### Detailed Feature Points

**3.1.1 Intelligent List Generation**
- **Based on prior year data**: Extracts required document types from the previous year's tax returns
- **Client persona-driven**: Generates different templates based on client type (individual/business/trust/nonprofit)
- **Dynamic adjustment**: Automatically updates request items based on current year's tax law changes
- **Multi-language support**: Supports generating request lists in English, Spanish, Chinese, and other languages

**3.1.2 List Customization**
- **Drag-and-drop editor**: Tax managers can drag to reorder request items
- **Conditional logic**: If client has rental income, automatically add Schedule E related requests
- **Annotation capability**: Add explanations for each request item (e.g., "Please provide all 2025 1099-INT forms")
- **Priority marking**: Flag documents as urgent/important/regular

**3.1.3 Automated Distribution**
- **One-click send**: Distribute via multiple channels including email, portal, and SMS
- **Personalized links**: Each client receives a unique upload link
- **Bulk operations**: Send annual PBC lists to all clients simultaneously

### 3.2 Intelligent Document Receipt and Classification Module

#### Function Description
Automatically receives, classifies, and organizes all uploaded tax documents.

#### Detailed Feature Points

**3.2.1 Multi-Channel Receipt**
- **Client portal upload**: Drag-and-drop upload, supports batch processing
- **Email forwarding**: Clients can forward documents to `upload@taxdox.ai`, system automatically associates with corresponding client
- **Mobile photo capture**: Mobile app photo upload, AI automatically optimizes image quality
- **Cloud import**: Import from Box, Google Drive, OneDrive/SharePoint
- **Batch ZIP upload**: Supports uploading entire folder ZIP archives

**3.2.2 AI Intelligent Classification**
Supports automatic identification of **100+ types of tax documents**:

| Category | Document Types |
|----------|----------------|
| **Income Documents** | W-2, 1099-NEC, 1099-MISC, 1099-INT, 1099-DIV, 1099-B, 1099-R, K-1(1065/1120S/1041) |
| **Deduction Documents** | Mortgage interest 1098, Property tax bills, Charitable donation receipts, Medical expense receipts, Education expense 1098-T |
| **Identity Documents** | Driver's license, Passport, SSN card, EIN confirmation letter |
| **Business Documents** | Profit and loss statements, Balance sheets, Bank statements, Credit card statements, Payroll reports |
| **Investment Documents** | Brokerage statements, Cryptocurrency transaction records, Dividend reinvestment plans |
| **Real Estate Documents** | Property purchase documents, Rental income records, Depreciation schedules, Property tax assessments |
| **Other** | Alimony agreements, Divorce decrees, Student loan 1098-E, Health Savings Account 5498-SA |

**3.2.3 Intelligent Renaming and Organization**
- **Automatic renaming**: Renames `IMG_20260315_142356.jpg` to `2025_W2_AcmeCorp.pdf`
- **Automatic filing**: Automatically organizes into folders by document type, year, and client
- **Version control**: New versions of the same document automatically overwrite old versions, with history retained
- **Duplicate detection**: Detects and flags duplicate document uploads

### 3.3 AI Data Extraction Engine

#### Function Description
Automatically extracts structured data from tax documents for import into tax software.

#### Detailed Feature Points

**3.3.1 Multi-Format Processing**
- **PDF**: Native PDF, scanned PDF, encrypted PDF
- **Images**: JPEG, PNG, TIFF, HEIC (mobile photos)
- **Handwriting recognition**: Recognizes handwritten notes and signatures
- **Excel**: Extracts data from Excel spreadsheets
- **Low-quality scans**: Enhances low-resolution, skewed, and blurry scans

**3.3.2 Field-Level Extraction**
Each document type has predefined extraction fields:

**W-2 Extraction Fields**:
- Employer EIN, Employer Name, Employer Address
- Employee SSN, Employee Name, Employee Address
- Box 1: Wages, tips, other compensation
- Box 2: Federal income tax withheld
- Box 3: Social Security wages
- Box 4: Social Security tax withheld
- Box 5: Medicare wages
- Box 6: Medicare tax withheld
- Box 7: Tip income
- Box 8: Allocated tips
- Box 10: Dependent care benefits
- Box 12: Code + Amount (multiple lines)
- Box 13: Retirement plan/Third-party sick pay/Statutory employee
- Box 14: Other (state disability insurance, etc.)
- State information: State name, State EIN, State wages, State tax withheld

**1099 Series Extraction Fields**:
- 1099-NEC: Payer information, Recipient information, Nonemployee compensation
- 1099-INT: Interest income, Early withdrawal penalty, Investment expenses
- 1099-DIV: Ordinary dividends, Qualified dividends, Capital gain distributions
- 1099-B: Transaction date, Cost basis, Sales proceeds
- 1099-R: Gross distribution, Taxable amount, Federal tax withheld, Code

**K-1 Extraction Fields** (supports 1065, 1120S, 1041):
- Entity information: EIN, Name, Address, Type
- Partner/Shareholder information: SSN/EIN, Name, Address, Ownership percentage
- Detailed allocations of income/deduction/credit sections
- Self-employment income, Net rental income, Foreign taxes, etc.

**3.3.3 Accuracy and Confidence**
- **Target accuracy rate**: 95%+ (MVP stage)
- **Confidence scoring**: Each extracted field includes a 0-100% confidence score
- **Low confidence flagging**: Fields with <90% confidence flagged for manual review
- **Continuous learning**: Model automatically improves after manual corrections

**3.3.4 Validation and Verification**
- **Cross-document validation**: Consistency checks between W-2 wages and 1099 income
- **Historical comparison**: Compares with prior year data, flags significant changes
- **Logic validation**: Checks if data conforms to tax law logic (e.g., donations not exceeding 60% of AGI)
- **Completeness checks**: Flags missing required documents

### 4.2 Data Model (Core Entities)

```sql
-- Core Entity Relationships
Users → Firms → Clients → Engagements → PBCLists → PBCItems → Documents → Extractions

-- Key Table Structures
CREATE TABLE firms (
  id UUID PRIMARY KEY,
  name VARCHAR(255),
  subscription_tier VARCHAR(50),
  settings JSONB,
  created_at TIMESTAMP
);

CREATE TABLE clients (
  id UUID PRIMARY KEY,
  firm_id UUID REFERENCES firms(id),
  name VARCHAR(255),
  email VARCHAR(255),
  tax_id VARCHAR(20),
  client_type VARCHAR(50), -- individual/business/trust/nonprofit
  metadata JSONB,
  created_at TIMESTAMP
);

CREATE TABLE engagements (
  id UUID PRIMARY KEY,
  client_id UUID REFERENCES clients(id),
  firm_id UUID REFERENCES firms(id),
  tax_year INTEGER,
  engagement_type VARCHAR(50), -- 1040/1065/1120/1120S/1041
  status VARCHAR(50),
  assigned_to UUID REFERENCES users(id),
  pbc_list_id UUID REFERENCES pbc_lists(id),
  deadline DATE,
  created_at TIMESTAMP
);

CREATE TABLE pbc_lists (
  id UUID PRIMARY KEY,
  engagement_id UUID REFERENCES engagements(id),
  name VARCHAR(255),
  is_template BOOLEAN,
  template_id UUID REFERENCES pbc_lists(id),
  created_at TIMESTAMP
);

CREATE TABLE pbc_items (
  id UUID PRIMARY KEY,
  pbc_list_id UUID REFERENCES pbc_lists(id),
  document_type VARCHAR(100),
  description TEXT,
  required BOOLEAN,
  priority VARCHAR(20), -- high/medium/low
  expected_format VARCHAR(50), -- pdf/jpg/xls
  extraction_template JSONB, -- defines fields to extract
  order_index INTEGER,
  created_at TIMESTAMP
);

CREATE TABLE documents (
  id UUID PRIMARY KEY,
  client_id UUID REFERENCES clients(id),
  engagement_id UUID REFERENCES engagements(id),
  pbc_item_id UUID REFERENCES pbc_items(id),
  original_filename VARCHAR(255),
  stored_filename VARCHAR(255),
  storage_path VARCHAR(500),
  file_size BIGINT,
  mime_type VARCHAR(100),
  document_type VARCHAR(100), -- AI classification result
  confidence FLOAT, -- AI classification confidence
  status VARCHAR(50), -- uploaded/processing/processed/reviewed/rejected
  uploaded_by VARCHAR(255), -- client_email or user_email
  uploaded_at TIMESTAMP,
  processed_at TIMESTAMP
);

CREATE TABLE extractions (
  id UUID PRIMARY KEY,
  document_id UUID REFERENCES documents(id),
  field_name VARCHAR(100),
  field_value TEXT,
  confidence FLOAT,
  source_location TEXT, -- position within document
  is_verified BOOLEAN, -- whether manually verified
  verified_by UUID REFERENCES users(id),
  verified_at TIMESTAMP,
  created_at TIMESTAMP
);

CREATE TABLE workflows (
  id UUID PRIMARY KEY,
  engagement_id UUID REFERENCES engagements(id),
  step VARCHAR(50),
  status VARCHAR(50),
  assigned_to UUID REFERENCES users(id),
  started_at TIMESTAMP,
  completed_at TIMESTAMP,
  metadata JSONB
);

CREATE TABLE audit_logs (
  id UUID PRIMARY KEY,
  firm_id UUID REFERENCES firms(id),
  user_id UUID REFERENCES users(id),
  action VARCHAR(100),
  resource_type VARCHAR(50),
  resource_id UUID,
  details JSONB,
  ip_address VARCHAR(45),
  user_agent TEXT,
  created_at TIMESTAMP
);
```
